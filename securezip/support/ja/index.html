はじめに確認すること

Visual Studio Code

SecureZipが最新版で、利用中のVisual Studio Codeにインストールされているか確認してください。

ワークスペース

ZIPにしたいフォルダーまたはワークスペースをVisual Studio Codeで開いているか確認してください。

除外ルール

.gitignore と .securezipignore の内容を確認してください。不要なファイルはこれらのルールで除外できます。

よくある質問

ZIPに含めたくないファイルがあります。

.gitignore または .securezipignore に除外したいパターンを追加してください。.securezipignore はSecureZip専用の除外ルールとして利用できます。

ZIPファイルが作成されません。

ワークスペースが開かれていること、保存先に書き込み権限があること、対象ファイルが除外ルールですべて除外されていないことを確認してください。

Auto CommitやTaggingは必須ですか。

必須ではありません。SecureZipの設定で、自動コミットやタグ付けの挙動を必要に応じて切り替えられます。

プロジェクトファイルは外部サーバーへ送信されますか。

送信されません。SecureZipのZIP作成処理はローカル環境で行われ、開発者が運用する外部サーバーへワークスペースデータを送信しません。

トラブルシューティング

  • Visual Studio CodeとSecureZipを最新版に更新してください。
  • Visual Studio Codeを再起動してください。
  • .gitignore と .securezipignore の除外ルールを確認してください。
  • 保存先フォルダーへの書き込み権限を確認してください。
  • Git関連機能で問題が出る場合は、対象フォルダーがGitリポジトリか、Gitが利用できる環境か確認してください。
