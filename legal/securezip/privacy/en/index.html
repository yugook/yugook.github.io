Privacy Policy

SecureZip (the "Extension") handles workspace information only as needed to create ZIP archives within Visual Studio Code.

Japanese version Legal index

1. Information We Access and Use

  • Workspace Data: To create ZIP archives, the Extension locally accesses file names, paths, and file contents inside the selected folder or workspace.
  • Ignore Rules: The Extension reads .gitignore, .securezipignore, and automatic exclusion rules to decide which files are included.
  • Git Data: If you use auto-commit or tagging, the Extension may read or update local Git repository status, commits, and tags.
  • Extension Settings: The Extension uses SecureZip settings stored in Visual Studio Code settings.

2. How We Use Information

Information is used to provide Extension features such as archive creation, ignore rule handling, previews, auto-commit, and tagging.

3. Data Transmission and Sharing

  • No Developer Server: The Extension does not send workspace data to servers operated by the developer.
  • No Advertising or Tracking: We do not sell data or use it for advertising tracking.
  • Platform Services: Data handling by Visual Studio Code, VS Code Marketplace, Git, or other tools in your environment is governed by those providers' terms.

4. Storage and Deletion

  • Created ZIP archives are saved to the destination selected by the user.
  • Git commits or tags are recorded in the local Git repository only when you use those features.
  • You can delete or change ZIP files, settings, and Git history in your own environment.

5. Security

The Extension performs archive creation locally and is designed to avoid handling more workspace data than needed.
